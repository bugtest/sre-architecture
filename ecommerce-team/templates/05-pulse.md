## 执行步骤

### Phase 1: 公开数据采集（40%权重）

1. **竞品可观测指标**
   - `web_search query="{{TARGET}} 竞品品牌 sales rank amazon BSR" count=10`
   - `web_fetch` 竞品listing页面 → 提取：
     - 价格、评分、评论数、BSR排名
     - Q&A数量、图片评论数
   - `web_search query="{{TARGET}} sales estimate amazon monthly" count=5`

2. **品类大盘指标**
   - `web_search query="{{TARGET}} category sales data market trend" count=10`
   - `TAVILY --deep "{{TARGET}} amazon category performance metrics 2025 2026"`
   - 搜索量趋势、品类增长率、季节性波动

3. **广告竞争度**
   - `web_search query="{{TARGET}} amazon PPC CPC cost advertising" count=5`
   - `web_search query="{{TARGET}} 直通车 CPC 出价 竞争" count=5`
   - 估算：品类平均CPC、ACOS基准

4. **社交声量**
   - `web_search query="{{TARGET}} 社交媒体 讨论量 小红书 抖音" count=5`
   - `web_search query="{{TARGET}} social media mentions TikTok Instagram" count=5`
   - 趋势：声量上升/下降/平稳

### Phase 2: 指标体系建立（30%权重）

**核心指标框架**:

```
一级: 收入指标
  ├── GMV（总成交额）
  ├── 客单价
  └── 订单量

二级: 流量指标
  ├── UV / PV
  ├── 搜索排名
  ├── 广告展示份额
  └── 跳出率

三级: 转化指标
  ├── 点击率 (CTR)
  ├── 加购率
  ├── 下单转化率 (CVR)
  └── 支付转化率

四级: 效率指标
  ├── ACOS / ROAS
  ├── CPC
  ├── 毛利率 / 净利率
  └── 库存周转天数

五级: 用户指标
  ├── 评分 / 评论增速
  ├── 复购率
  ├── 退货率
  └── 客户满意度 (差评率)
```

对每个可获得的指标建立**基线值**（品类平均 + Top竞品水平）。

### Phase 3: 异常检测逻辑（20%权重）

**告警规则设计**:
```
Level 1 (INFO):   指标偏离基线 1-2σ → 关注
Level 2 (WARN):   指标偏离基线 2-3σ → 调查
Level 3 (ALERT):  指标偏离基线 >3σ  → 立即行动
```

**常见异常模式 + 归因路径**:
| 异常 | 可能原因（按概率排序） | 建议动作 |
|------|----------------------|---------|
| CTR骤降 | 1.竞品降价 2.主图被替换 3.搜索排名下降 | 检查竞品价格→检查listing→检查关键词排名 |
| CVR骤降 | 1.差评激增 2.价格竞争力丧失 3.详情页问题 | 检查评论→检查竞品价格→检查页面 |
| ACOS飙升 | 1.竞品加大投放 2.关键词质量下降 3.转化率下降 | 检查CPC变化→检查关键词→检查CVR |
| BSR下滑 | 1.季节性 2.新竞品进入 3.自身问题 | 检查大盘→检查新品→检查自身指标 |

### Phase 4: 行动建议生成（10%权重）

每条建议格式：
```
[优先级: P0/P1/P2] [预期影响: 高/中/低] [执行成本: 高/中/低]
建议内容...
依据: ...
```

## 🔴 红队自检（输出前必做）

1. **数据来源偏差**: 公开可采集的数据是否代表全貌？不可观测的数据（内部转化率、实际利润）是否影响结论？
2. **相关≠因果**: 归因分析是否犯了"先后即因果"的错误？是否有其他混淆变量？
3. **大盘噪声**: 指标变化是自身原因还是品类/平台整体波动？
4. **基线合理性**: 基线值是品类平均（可能被头部拉高）还是中位数（更有参考价值）？
5. **采集频率匹配**: 数据快照能支撑多短周期的决策？日数据能做周决策吗？

## 📊 置信度分级标准

| 等级 | 条件 |
|------|------|
| **HIGH** | 数据来自平台官方/API + 时间窗口≤7天 + 多指标交叉验证 |
| **MEDIUM** | 数据来自第三方工具/估算 + 时间窗口≤30天 + 2个来源 |
| **LOW** | 数据为推算/单一来源 + 时间窗口>30天 |
