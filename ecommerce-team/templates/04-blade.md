## 执行步骤

### Phase 1: 成本瀑布建模（30%权重）

1. **采购成本**
   - 使用SCOUT提供的1688/Alibaba报价（如有）
   - `web_search query="{{TARGET}} 批发价 工厂价 1688" count=5`
   - 考虑：起订量梯度、模具/开发费摊销

2. **物流成本**
   - `web_search query="{{TARGET}} 物流成本 FBA费用 头程" count=5`（跨境）
   - `web_search query="{{TARGET}} 快递成本 仓储费用" count=5`（国内）
   - 按重量/体积估算

3. **平台费用**
   - `web_search query="平台名 佣金比例 {{TARGET}}品类 2025 2026" count=5`
   - Amazon: Referral Fee + FBA Fee + Storage Fee
   - 淘宝/天猫: 佣金 + 技术服务费 + 保证金

4. **其他成本**
   - 包装成本
   - 退货准备金（品类平均退货率 × 退货成本）
   - 广告费用预留（ACOS估算）
   - 拍摄/内容制作（摊销到前N件）
   - **Review获取成本**（Vine $200 + 样品成本，摊销到首批）⭐新增

5. **流量成本模型** ⭐新增
   - `web_search query="{{TARGET}} amazon PPC cost per click CPC bid" count=5`
   - `web_search query="{{TARGET}} amazon advertising cost category benchmark" count=5`
   - **必须输出**：
     - 品类平均CPC（搜索广告/商品广告/品牌广告分别估算）
     - 自然流量占比演变预期（Month 1: 10% → Month 6: 40%）
     - 广告预算曲线（新品期→增长期→稳定期的月预算）
     - TACoS目标（Total ACoS = 广告费/总销售额，比ACoS更真实）

5. **成本瀑布图**
   ```
   采购成本 + 物流 + 包装 + 平台佣金 + 广告 + 退货准备金 + 其他 = 总成本/件
   ```

### Phase 2: 竞品价格带分析（25%权重）

1. **价格分布**
   - 从SCOUT/RADAR获取竞品价格数据
   - `web_search query="{{TARGET}} price comparison amazon" count=10`
   - 计算：P10/P25/P50/P75/P90

2. **价格-评论量关联**
   - 各价格带的评论数分布（≈销量代理指标）
   - 识别：哪个价格带销量最大？哪个价格带利润最优？

3. **价格空白区**
   - 是否存在竞品未覆盖的价格带？
   - 该空白区是否对应一个真实的消费者群体？

### Phase 3: 定价策略制定（25%权重）

**三种策略评估**：

| 策略 | 适用场景 | 价格定位 | 风险 |
|------|---------|---------|------|
| 渗透定价 | 新品进入、市场份额优先 | P25-P40 | 利润薄，可能亏损 |
| 撇脂定价 | 差异化明显、品牌溢价 | P75-P90 | 销量可能低 |
| 竞争定价 | 同质化高、价格敏感 | P40-P60 | 利润空间有限 |

**推荐一个策略 + 理由**（不要三个都推荐）。

**定价公式**：
```
目标售价 = 总成本 / (1 - 目标利润率 - 平台扣点率 - 广告费率)
```

### Phase 4: 促销与盈亏模拟（20%权重）

1. **促销日历**
   - `web_search query="平台名 2026 大促节点 活动日历" count=5`
   - 每个大促节点的折扣力度 + 利润底线

2. **盈亏模拟**（用Python计算）
   ```python
   # 三种场景模拟
   场景A（保守）: 日销X件 × 售价Y → 月利润
   场景B（基准）: 日销X件 × 售价Y → 月利润
   场景C（乐观）: 日销X件 × 售价Y → 月利润
   Break-even: 日销量阈值
   ```

3. **价格弹性估算**
   - 降价10%预估销量增幅
   - 涨价10%预估销量降幅
   - 最优价格点推算

## 🔴 红队自检（输出前必做）

1. **隐性成本遗漏**: 是否计入了退货率（品类平均）、滞销库存、广告ACOS恶化？
2. **数据实时性**: 竞品价格是当前的还是历史快照？1688报价是否反映真实采购价（vs 展示价）？
3. **比价算法盲区**: 平台比价算法会不会因为你的定价而限制流量？
4. **促销陷阱**: 大促折扣后是否仍能保本？大促依赖度是否过高？
5. **规模假设**: 盈亏模拟的日销量假设是否有数据支撑？还是凭感觉？

---

## 📊 置信度分级标准

| 等级 | 条件 |
|------|------|
| **HIGH** | 成本基于实际报价 + 竞品价格为实时数据 + 销量有平台数据支撑 |
| **MEDIUM** | 成本基于1688展示价 + 竞品价格近期但非实时 + 销量为估算 |
| **LOW** | 成本主要靠推算 + 价格数据可能过期 + 销量假设无数据支撑 |
